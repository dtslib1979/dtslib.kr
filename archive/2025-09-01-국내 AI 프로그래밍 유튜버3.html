요약(Executive Summary)

  • YouTube는 튜토리얼(How-to) 중심. ROI가 가장 큰 분야는 AI 프로그래밍.
  • 핵심 루틴: 예습 → 시청 → 실습 → 정리 → 응용. 예습은 AI 요약(Gemini/ChatGPT), 실습은 PC 기반.
  • 크리에이터 전략: 코드팩토리를 메인으로 집중. 조코딩은 트렌드 참고, 잔재미코딩은 데이터 기초 보충.
  • 환경 원칙: PC = 개발/빌드, 태블릿 = 보조(노트, 질의응답, 검토).

크리에이터 성장 이력 & 현재 포지션

조코딩 (JoCoding)

  • 초기: 흥미 프로젝트(노코드, 간단 AI 웹)로 대중성 확보.
  • 중기: 트렌드 실험 확장 → 뉴스/엔터테인먼트 성격 강화.
  • 현재: 빠른 성취감 제공 장점. 한계는 깊이·체계성.

코드팩토리 (Code Factory)

  • 초기: Flutter 중심 실무 튜토리얼로 주니어 흡수.
  • 중기: Flutter+NestJS+풀스택(Full-stack)로 채용 수요 직격.
  • 현재: 실무형 구조/클린 코드, 포트폴리오 직결. 초심자 진입장벽은 AI 보조로 완화.

잔재미코딩 (Janjaemi Coding)

  • 초기: Python 기초·Pandas·Numpy로 입문자 유입.
  • 중기: 머신러닝/딥러닝(ML/DL) 기초까지 A to Z.
  • 현재: 학술적 체계는 강점, 실무 연결·속도는 약점.

공통분모(Overlap)만 뽑은 최소 커리큘럼

  • Python 기초(variables, functions, control flow, file I/O).
  • API 활용(REST API, OpenAI API)과 서비스 연결(backend ↔ frontend).
  • 데이터 입출력·가공(CSV/JSON, Pandas 기초).
  • 실습 중심(worked example) 루틴: 따라치기 → 변형 → 오리지널.

한 줄 정의: Python × API × Data × Project

학습 루틴 (5단계)

예습(Preview) — URL을 AI에 넣고 초심자 수행 단계 요약 받기.
시청(Watch) — 타임라인 키워드 기록, 모르는 개념 즉시 질의.
실습(Practice) — PC에서 동일 환경(Colab/VSCode, SDK, API key)으로 완벽 복사.
정리(Archive) — 기능/개념/에러해결을 “내 언어”로 기록(Samsung Notes/Blog/GitHub).
응용(Extension) — 색/데이터/기능 변형 → 오리지널 프로젝트.

환경 전략(PC/Tablet)

  • PC: VSCode, Flutter SDK, Node.js, DB, GitHub — 빌드/런(빌드·배포) 필수.
  • 태블릿: 예습 요약, 노트, 코드 검토, AI 질의응답 — 보조(Console/Viewer) 역할.
  • ML/DL 학습(TensorFlow/PyTorch)은 연산·환경 제약으로 태블릿 단독 불가 → 클라우드(Colab, Codespaces) 연결로만 보완.

최종 선택(Selection)

  • 메인: 코드팩토리 — 실무형 프로그래밍·앱/웹 편집·활용 중심.
  • 보조: 조코딩(트렌드 레퍼런스), 잔재미코딩(데이터 기초 보충).
  • 버림: 온디바이스 ML/DL 학습·이론 심화 — 필요·흥미·환경 모두 불일치.

자주 묻는 질문(FAQ)

왜 한 채널(코드팩토리) 집중이 유리한가?
  • 커리큘럼 연속성 유지로 누적 학습이 된다.
  • 포트폴리오 직결 산출물 확보가 쉽다.
  • AI 보조(ChatGPT)로 난이도·속도 보정이 가능하다.
태블릿만으로 가능한 최대치는?
  • 브라우저 기반 실습(Colab, Codespaces) 접속, 기록·요약·리뷰.
  • 경량 추론(TFLite/CoreML) 테스트 수준. 학습(Training)은 클라우드/PC 필요.
최소 커리큘럼만으로도 충분한가?
  • Python 기초 + API + Data I/O + 프로젝트 반복이면 실용 목적에는 충분.
  • 대규모 시스템 설계/CS 이론은 필요시 별도 보강(CS50 등) 권장.

가설 리스트(User-Only Ideas)

  • YouTube는 튜토리얼(How-to) 매체이므로, AI 프로그래밍만 집중하면 ROI가 최고다.
  • 코드팩토리 단일 집중이 실무·포트폴리오 달성에 가장 빠르다.
  • 과도기 하이브리드(PC 메인 × 태블릿 보조)가 현실적 최적점이다.
  • “PC 전용 강의의 태블릿 재현”을 별도 프로젝트화하면 교육 콘텐츠로 변환될 수 있다.
  • 학습은 예습(요약)–시청–실습–정리–응용의 고정 루틴으로 표준화 가능하다.
  • ML/DL 이론·학습은 의도적으로 배제해도 실용 목표에는 지장 없다.
  • YouTube × AI × 실습 루틴은 개인에게 “1인 대학(One-Person University)”가 될 수 있다.

전문가 평가 칼럼(Expert Review)

강점: 목표(편집·활용)와 수단(코드팩토리 집중, AI 예습·디버깅)이 정확히 정렬되어 있다. 최소 커리큘럼(Python·API·Data·Project)으로 낭비를 없앴고, PC/태블릿 역할 분리로 실행 가능성을 극대화했다. 인포그래픽은 JS 없이 SVG로 구성되어 이식성과 가볍기가 좋다.

보강점: 장기 유지 보수를 고려해 버전 고정(version pinning)과 환경 스냅샷(devcontainer/Codespaces)을 추가하라. 또한 포트폴리오 각 항목에 문제정의–해결–성과 3줄 요약을 붙여 검색·면접 대비력을 높이면 좋다.
